# Pack para Codex

Contenido:
- `AGENTS.md`: reglas permanentes del repositorio.
- `PLANS.md`: plan por fases y sprints.
- `PROMPT_SPRINT_1.txt`: orden puntual para ejecutar el Sprint 1.

Uso recomendado:
1. Descomprime este paquete en la raíz del repositorio donde quieres que trabaje Codex.
2. Verifica que `AGENTS.md` y `PLANS.md` queden visibles en la raíz.
3. Abre Codex en esa carpeta.
4. Pega el contenido de `PROMPT_SPRINT_1.txt` como primera orden.

Objetivo:
Construir un runtime orquestador de proyectos con tareas pequeñas, persistencia, validación real, recovery y benchmarks.
