Cómo funciona LACE en la realidad: explicación técnica de cómo un agente LLM procesa el archivo .md

1
Tú le das al agente dos cosas al mismo tiempo
Cuando usas Claude Code o Codex en una terminal, el agente recibe el prompt del humano y también puede leer los archivos del proyecto. Son dos inputs separados que el agente combina.
Prompt del humano: "Crea un sistema de registro de procesos con HTML, JS y base de datos"

Archivo en el proyecto: LACE.md — las instrucciones de comportamiento
2
El agente lee el .md y lo convierte en contexto
Un LLM como Claude o GPT-4 no "ejecuta" el archivo como si fuera código. Lo lee como texto y ese texto pasa a formar parte de su contexto — su "memoria de trabajo" para esa sesión. Es exactamente igual a cuando tú lees un manual antes de operar una máquina.
# Lo que el agente procesa internamente:
contexto = prompt_del_humano + contenido_de_LACE.md
# Ahora el agente "sabe" las reglas antes de actuar
3
El agente combina la tarea real con las reglas del .md
Aquí está la magia. El agente no hace la tarea ignorando el LACE.md. Hace la tarea a través del filtro de las reglas. Es como darle a un cocinero una receta (el prompt) más un manual de calidad (el LACE.md). El cocinero cocina la receta, pero con los estándares del manual.
Sin LACE.md: "Crear sistema de registro" → lo crea → cierra → espera al humano

Con LACE.md: "Crear sistema de registro" → lo crea → lee las reglas → se pregunta: ¿cumplí los 10 ciclos? No → vuelve a analizar → mejora → vuelve a preguntar → ...
4
¿Cómo "obliga" el archivo al agente? — la clave real
El .md no es código que se ejecuta. Es lenguaje natural muy claro y directo. Los LLMs son extremadamente buenos siguiendo instrucciones escritas en texto cuando estas son específicas. El LACE.md dice cosas como:
"No puedes cerrar este proyecto hasta completar 10 ciclos"
"Después de cada ciclo pregúntate: ¿mejoró? Si no, repite"
"Documenta cada decisión en LACE_LOG.md"
El agente lee esas frases, las interpreta, y las convierte en comportamiento. No porque haya código que lo fuerce — sino porque su entrenamiento le enseñó a seguir instrucciones de texto con precisión.
5
Resultado: el agente hace la comparación internamente
En cada ciclo, el agente compara lo que tiene ahora contra los criterios del LACE.md. Esa comparación no es magia — es predicción de texto. El agente predice qué texto viene después de "analicé el proyecto y encontré que..." y esa predicción lo lleva a identificar deficiencias.
Ciclo 1: Proyecto creado. LACE dice: analiza. → "El frontend es básico, falta validación"
Ciclo 2: Mejoré frontend. LACE dice: critica. → "Falta multiusuario, manejo de errores"
Ciclo 3: Agregué errores. LACE dice: valida. → "Funciona. ¿Cumplí 10 ciclos? No. Sigo."
...
Ciclo 10: Todo mejorado. LACE dice: evalúa criterios finales. → "Sí. Cierro."
La respuesta corta a tu pregunta: el LACE.md no es una guía metafórica para ti el humano. Es un conjunto de instrucciones de texto que el agente lee, interpreta y sigue como si fueran órdenes directas — porque para un LLM, leer instrucciones claras en texto es exactamente lo mismo que recibir órdenes. El archivo convierte al agente en un trabajador que no puede cerrar su turno hasta completar el checklist.
