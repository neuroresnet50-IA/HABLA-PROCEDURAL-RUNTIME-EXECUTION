Fase 0 — arranque dirigido por .md
El agente NO empieza a ciegas. Lee el archivo LACE.md que contiene el mapa completo de lo que debe hacer. Sin ese archivo, el agente no actúa.
# LACE.md — instrucciones globales
## obligaciones del agente
- ejecutar mínimo 5 ciclos completos
- no cerrar sin pasar la puerta de calidad
- documentar cada decisión en LACE_LOG.md
## orden de ejecución: [0] leer → [1] crear → [2..5] iterar
1
Fase 1 — creación base (plan A)
El agente construye la versión inicial del proyecto. Cuando termina, NO cierra. En cambio, activa automáticamente el primer ciclo LACE.
2
Fases 2–6 — ciclos LACE (mínimo 5 iteraciones)
Cada ciclo tiene exactamente 4 pasos en orden fijo. El agente no puede saltarlos.

Paso 1 — Analizar
7
Puerta de calidad — ¿cerrar o iterar más?
Después de 5 ciclos, el agente evalúa contra los criterios del LACE.md. Si no pasa, inicia otro ciclo. No hay límite superior excepto el que define el humano.
¿Pasó todos los criterios? → Cerrar  |  ¿Algo falló? → Ciclo adicional
✓
Cierre — entrega con LACE_LOG.md
El agente cierra entregando el proyecto + un log completo de cada decisión, cada mejora y cada razón. El humano nunca ve solo código — ve el razonamiento.
El nombre clave del comportamiento que describes: iteración forzada con autocrítica estructurada. Lo que diferencia LACE de una ejecución normal es que el agente tiene prohibido cerrar antes de completar los ciclos, y cada ciclo tiene que producir algo diferente al anterior.
